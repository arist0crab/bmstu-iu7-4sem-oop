## Немного теории

> [!NOTE]
> Контейнерный класс (класс-контейнер) — это специальный шаблонный тип класса, предназначенный для хранения, организации и управления группой объектов одного типа. Он предоставляет унифицированные методы для добавления, удаления и доступа к элементам, скрывая сложную логику работы с памятью. 

Перед тем, как приступить непосредственно к реализации, рекомендуется выполнить четыре квеста:

1. Прослушать соответствующие лекции по ООП
2. Прочитать про [основы управления ресурсами](https://en.cppreference.com/w/cpp/language/rule_of_three.html) ([русская версия](https://habr.com/ru/companies/otus/articles/704492/)). Сюрприз-сюрприз: все 4 класса по заданию [владеют ресурсами](https://myrusakov.ru/cpp-raii-simple-guide.html), поэтому нам точно не будет скучно. Также настоятельно рекомендуется изучить [copy-and-swap idiom](https://stackoverflow.com/questions/3279543/what-is-the-copy-and-swap-idiom)
3. Изучить [документацию](https://en.cppreference.com/w/cpp/named_req/Container) на тему "что делает контейнер контейнером", которая определяет часть списока методов, которые необходимо будет реализовать
4. Обратиться к теме [перегрузки операторов](https://habr.com/ru/articles/132014/) ([тут](https://learn.microsoft.com/ru-ru/cpp/cpp/operator-overloading?view=msvc-170) страничка Microsoft)

## Разбор общего условия и Формализованная Постановка Задачи

Попытаемся немного расширить условие от "сделай хорошо лабораторную работу, а плохо делать не надо" до ФПЗ, опираясь на приведенные выше материалы, а также на комментарии сдающих прошлых, текущих и будущих лет.

### 1. Управление ресурсами и памятью

**1.1.** Для хранения элементов **запрещено** использование стандартных контейнеров STL (`std::vector`, `std::list`, `std::deque` и т.п.). Внутреннее хранилище данных должно быть реализовано вручную.

**1.2.** Управление динамической памятью должно осуществляться с помощью **умных указателей** 

**1.3.** Класс должен корректно управлять ресурсами в соответствии с **Правилом пяти**:
- Деструктор
- Конструктор копирования
- Оператор присваивания копированием
- Конструктор перемещения
- Оператор присваивания перемещением

**1.4.** Для обеспечения безопасности при присваивании **обязательно** применение идиомы **Copy-and-Swap** в операторе присваивания.

### 2. Шаблоны, концепты и метаданные

**2.1.** Класс является шаблонным. Параметр шаблона `T` — тип хранимых элементов.

**2.2.** Параметр шаблона `T` должен быть ограничен концептом. Концепт должен проверять минимально необходимые требования для работы контейнера (копируемость, разрушаемость).

**2.3.** Концепт должен проверяться с помощью `static_assert` внутри класса для ранней диагностики ошибок компиляции.

**2.4.** Для совместимости со стандартной библиотекой в классе должны быть объявлены следующие **алиасы**:
- `value_type` — `T`
- `reference` — `T&`
- `const_reference` — `const T&`
- `size_type` — тип для размеров (обычно `std::size_t`)
- `difference_type` — тип для разности итераторов (обычно `std::ptrdiff_t`)
- `iterator` — тип итератора
- `const_iterator` — тип константного итератора
- `reverse_iterator` — `std::reverse_iterator<iterator>`
- `const_reverse_iterator` — `std::reverse_iterator<const_iterator>`

### 3. Циклы, операторы и итераторы

**3.1. Итераторы**

Реализовать **полный набор итераторов**:
- `iterator` — итератор произвольного доступа 
- `const_iterator` — константный итератор произвольного доступа
- `reverse_iterator` — обратный итератор 
- `const_reverse_iterator` — константный обратный итератор

Требуемые методы для работы с итераторами:
- `begin()`, `end()`
- `cbegin()`, `cend()`
- `rbegin()`, `rend()`
- `crbegin()`, `crend()`

**3.2. Операторы**

Перегрузить следующие операторы. Каждый перегруженный оператор **обязательно дублировать именованной функцией**:

| Оператор | Именованная функция |
|----------|---------------------|
| `==` | `equal` |
| `!=` | `not_equal` |
| `<` | `less` |
| `<=` | `less_equal` |
| `>` | `greater` |
| `>=` | `greater_equal` |
| `[]` (доступ по индексу) | `at` |
| `=` (присваивание) — только оператор, именованная функция не требуется |

**3.3.** В реализации методов, связанных с обходом элементов, **рекомендуется** использовать `std::ranges` вместо явных циклов `for`/`while`.

### 4. Конструкторы

Класс должен предоставлять следующие конструкторы:

| Конструктор | Описание |
|-------------|----------|
| `Container()` | Конструктор по умолчанию (создаёт пустой контейнер) |
| `Container(size_type count)` | Создаёт контейнер с `count` элементами, инициализированными по умолчанию |
| `Container(size_type count, const T& value)` | Создаёт контейнер с `count` элементами, инициализированными значением `value` |
| `Container(std::initializer_list<T> list)` | Конструктор от списка инициализации |
| **`template<typename Container>`<br>`Container(const Container& other)`** | Шаблонный конструктор от **любого** контейнера, поддерживающего `begin()`/`end()` (включая `std::vector`, `std::list`, `std::set` и т.д.) |
| `Container(const Container&)` | Конструктор копирования |
| `Container(Container&&)` noexcept | Конструктор перемещения |

**Важно:** шаблонный конструктор от любого контейнера не должен перехватывать вызовы других конструкторов. Для этого используется `enable_if` или концепты, проверяющие, что тип не является текущим контейнером и имеет итераторы.

### 5. Исключения и обработка ошибок

**5.1.** Методы, которые **не могут бросить исключение**, должны быть помечены `noexcept`. К ним относятся:
- Деструктор
- `swap()`
- `size()`, `empty()`
- `begin()`, `end()` и их константные/обратные версии
- Конструктор перемещения
- Оператор присваивания перемещением

**5.2.** Методы, которые могут бросить исключение, должны оставлять объект в **валидном состоянии**. Для этого:
- Использовать **идиому Copy-and-Swap** в операторах присваивания
- В конструкторах использовать списки инициализации с обработкой ошибок
- При невозможности выделить память допускается исключение `std::bad_alloc`

**5.3.** В случае выхода за пределы контейнера (оператор `[]`, функция `at()`) допускается неопределённое поведение для `operator[]` и выбрасывание исключения `std::out_of_range` для функции `at()`.

**5.4.** Для ранней проверки корректности типов использовать `static_assert` с концептами
